# # 3주차: 프로젝트 구조 및 역할·일정 관리 계획 

1주차 프로젝트 개요, 2주차 요구사항 분석 및 설계를 바탕으로, 3주차에는 프로젝트를 실제로 수행하기 위한 구조, 역할, 일정 관리 계획을 수립합니다.

## ## 1. 프로젝트 구조 설계

이전 주차에 도출한 기능적 요구사항(F-01~F-15)을 바탕으로, 3계층 분리 구조(View -> Service -> Storage) 중 본인이 담당하는 백엔드(Service, Storage)를 중심으로 시스템 구조를 명세화했습니다.

### ### 1.1 기능/화면/시스템 구조 정리 
* **기능 구조 (Backend):**
    * **플레이 로직 관리:** 입력 감지(F-01), 랜덤 이벤트 트리거(F-02), 사망 판정 및 0.1초 오차 제어(F-03), 난이도 가속 로직(F-05)
    * **데이터 처리 및 분석:** 실시간 점수 계산(F-04), CSV 파일 입출력 및 데이터 동기화(F-08, F-10), Java Stream API 기반 랭킹 정렬 및 통계(F-09)
    * **유효성 및 제어:** 고유 ID 발급(F-07), 게임 일시정지/재개(F-06), 결과 리포트 생성(F-14), 재시작 및 종료(F-15)
* **화면 구조 (Frontend):** 픽셀 아트 메인 GUI, 상단 실시간 점수 UI, 이벤트 발생 1초 전 테두리 점멸(Red Glow) 시각 경고(F-11), 캐릭터 애니메이션(F-12), 사운드 연동(F-13)
* **시스템 데이터 흐름:** `사용자 입력(View)` -> `데이터 검증 및 게임 비즈니스 로직(Service)` -> `로컬 CSV 영구 저장(Storage)`

### ### 1.2 주요 구성 요소 및 디렉토리 구조
요구사항 기능 요소가 각 클래스에 어떻게 배분되어 있는지 도식화한 디렉토리 구조입니다.

## ## 2. 팀 역할 분담

프로젝트의 효율적인 진행을 위해 계층별로 명확히 담당 업무를 분배하였으며, 원활한 협업 프로세스를 구축했습니다.

### ### 역할 정의 원칙
* 각 팀원이 명확한 담당 모듈을 갖도록 분배
* 공통 작업(README, .gitignore, 통합 테스트)은 팀 전체 책임

### ### 2.1 팀원별 담당 업무와 책임 범위
| 팀원 | 담당 역할 | 주요 작업 및 책임 범위 |

| **국정원** | **백엔드 / 데이터** | - `Player` 객체 모델링 및 고유 ID 자동 발급<br>- `CsvStorage` 로컬 파일 입출력 및 예외 처리 동기화<br>- Java Stream API 활용 데이터 필터링 및 게임 엔진 로직 처리 |
| **엄태준** | **프론트엔드 / 총괄** | - Java Swing 기반 `MainFrame` GUI 화면 렌더링<br>- 사용자 입력 1차 예외 처리 및 애니메이션 시각 효과 구현<br>- 전체 시스템 기능 통합 및 시나리오 테스트 진행 |

### ### 2.2 협업 및 의사소통 방안 (GitHub 활용)
* **이슈 트래킹 (Issues):** 2주차 명세에 따라 모든 요구사항(F-01~F-15)을 기능 단위로 등록하고 **Assignee** 기능을 통해 담당 소관 명확화
* **진척도 시각화 (Projects):** 칸반 보드(Board)를 활용해 `Todo` -> `In Progress` -> `Done` 상태로 실시간 작업 현황 공유
* **코드 리뷰 및 병합 (Pull Request):** 기능 완성 시 PR을 생성하고, 상호 코드 리뷰를 필수로 진행하여 안전하게 `main` 브랜치에 병합

## ## 3. 일정 관리 계획

### ### 3.1 주차별 세부 일정 (WBS)
| 주차 | 작업 내용 | 담당 | 비고 |

| **1주차** | 프로젝트 기획, 요구사항 분석 및 시스템 아키텍처 설계 | 전체 | 완료 |
| **2~3주차** | **핵심 기능 구현:** CSV 데이터 I/O 로직 구축, 기본 루프 및 UI 뼈대 작성 | 역할별 | **Milestone 1** 달성 |
| **4주차** | **심화 및 통합:** Stream API 통계 로직 연동, 시각 효과 적용, 통합 테스트 | 전체 | **Milestone 2** 달성 |
| **5주차** | **최종 마감:** 코드 리팩토링, 안정성 점검, 문서화(README) 마감 및 제출 | 전체 | 배포 및 제출 |

### ### 3.2 마일스톤 및 중간 점검 계획
각 마일스톤 달성 시 다음 항목들을 집중적으로 점검하여 프로젝트의 완성도를 높입니다.

* **Milestone 1 (핵심 데이터 로직 점검 )**
    * **점검 내용:** 백엔드 테스트를 통해 데이터(점수/기록)가 로컬 CSV 파일에 권한 오류 없이 실시간으로 쓰기/읽기가 되는지, 초기 빈 파일 생성 예외 처리가 올바르게 작동하는지 확인
* **Milestone 2 (통합 연동 및 UI 테스트 점검 )**
    * **점검 내용:** 백엔드에서 Stream API로 집계한 기록 통계 수치가 화면 대시보드에 정확히 출력되는지 검증
    * **유효성 검사:** 잘못된 키 입력이나 시스템 예외 발생 시 서비스 계층에서 차단 로직이 정상 작동하는지 1:1 교차 검증
